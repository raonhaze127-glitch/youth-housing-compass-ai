# 에이전트 구조

## 현재 구현 수준

현재 MVP의 에이전트 구조는 처리 역할을 설명하기 위한 논리 모델입니다. 실제 구현은 규칙 기반 함수와 구조화된 샘플 데이터로 동작하며, 서로 독립적으로 판단하거나 외부 도구를 사용하는 다중 에이전트 시스템은 아닙니다.

## 에이전트 1: 청년주거전문상담가

- 사용자 질문 이해
- 나이, 지역, 무주택 여부, 소득 수준, 관심 유형 추출
- 조건이 부족하면 추가 질문 유도

현재는 `lib/parser`의 규칙 기반 조건 추출까지 구현되어 있습니다. 조건이 부족할 때 자동으로 후속 질문을 이어가는 기능은 향후 확장 범위입니다.

## 에이전트 2: 공고분석 에이전트

- 현재는 구조화된 `housing_programs.json` 데이터를 활용
- 향후 k-apt-alert 기반 공고 원문 수집 및 분석으로 확장
- 공고 원문에서 자격요건, 신청기간, 지역, 공급대상, 준비서류 추출 예정

현재 데이터는 공모전 시연용 샘플이며 실시간 수집 결과가 아닙니다.

## 에이전트 3: 추천 에이전트

- 사용자 조건과 공고 데이터를 매칭
- 접수중, 모집예정, 마감 상태를 고려해 정렬
- 추천 이유와 확인 필요사항 제공

현재는 `lib/matcher`, `lib/status`, `lib/recommender`에 역할을 분리했습니다. 추천은 참고용이며 실제 자격 판정을 대신하지 않습니다.

## 역할 연결

`사용자 입력 → 청년주거전문상담가 → 공고분석 데이터 → 추천 에이전트 → 참고용 결과`
