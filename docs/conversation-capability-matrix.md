# 청주나 대화형 상담 지원 범위

## 실행 흐름

모든 질문은 Orchestrator가 의도를 분류하고 담당 Agent에 전달한다. 담당 Agent의 답변은 Verification Agent가 금지 표현, 과도한 확신과 공고 원문 확인 안내를 검사한 뒤 반환한다.

API 응답에는 다음 추적 필드가 포함된다.

- `intent`: 분류된 질문 유형
- `handledBy`: 답변 담당 Agent
- `handledByLabel`: 화면 표시 이름
- `agentTrace`: 호출된 Agent 순서
- `verification`: 검증 상태와 수행한 검사

## 지원 질문

| 범위 | 담당 Agent | 가능한 질문 예시 | 현재 답변 수준 |
|---|---|---|---|
| 정책·용어 | Policy Agent | “공공임대와 공공분양 차이는?”, “국민임대가 뭐야?”, “임대주택 종류 알려줘” | KB 기반 정의·유형·차이 설명 |
| 사전 자격 | Eligibility Agent | “제가 신청 가능할까요?”, “소득 기준은?”, “1번 자격?”, “가점 계산해줘” | 입력값과 공고 근거를 비교한 참고 진단. 공고 배점표 미확인 시 계산 보류 |
| 공고 해석 | Announcement Agent | “1번 준비서류?”, “언제 마감?”, “보증금은?”, “원문 링크?”, “경쟁률은?” | 구조화된 일정·서류·임대조건·링크 설명. 없는 값은 확인 필요 처리 |
| 맞춤 추천 | Recommendation Agent | “서울 28세 무주택 청년 공고 추천해줘” | 지역·연령·무주택·대상·일정 기반 우선 검토 순서 |
| 안전 검증 | Verification Agent | 외부에서 직접 호출하지 않음 | 모든 답변의 근거·확신 표현·원문 안내 검사 |

## 입력이 충분할 때 가능한 진단

- 지역·시군구에 맞는 공고 후보 축소
- 나이 상·하한과 청년 전용 공고의 명백한 불일치 제외
- 무주택 필수 공고와 입력값 비교
- 신혼부부·청년·다자녀 등 대상계층 비교
- 공고 신청기간과 현재 상태 설명
- 공고에 구조화된 소득·자산 조건과 준비서류 안내
- 공고 유형과 공급 유형이 확인된 경우에만 배점 검토

## 아직 제한되는 질문

- 정확한 소득·자산 판정: 현재 프로필 입력이 구간 중심이라 가구원별 기준금액 계산은 추가 구현이 필요하다.
- 최종 신청 자격: 행정정보·제출서류 검증 권한이 없어 확정할 수 없다.
- 당첨 확률과 당첨 보장: 공식 경쟁률·배점 분포가 없으며 보장 표현을 사용하지 않는다.
- 법률 분쟁·이의신청의 최종 판단: 공식 기관 또는 전문가 확인이 필요하다.
- 실제 청약 신청·서류 제출·결제 대행: 수행하지 않는다.
- 날씨, 일반 뉴스, 민간 매매·투자 상담 등 공공주택 상담 밖의 질문.
