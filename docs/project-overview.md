# 프로젝트 개요

## 한 줄 소개

청년주거나침반 AI는 사용자가 입력한 주거 조건을 구조화하고, LH·SH·GH 및 청약홈 국민주택 모집공고와 비교해 참고할 공고를 보여주는 공모전 출품용 MVP입니다.

## 현재 구현

- 한 번의 자연어 문장을 받는 대화형 입력 UI
- 규칙 기반 나이, 지역, 무주택 여부, 소득 수준, 관심 유형 추출
- 저장 실공고 우선 추천과 `housing_programs.json` 최종 폴백
- 추천 이유, 신청기간, 지원내용, 원문 링크를 담은 결과 카드
- 신청기간을 기준으로 계산한 접수중, 모집예정, 마감 상태 표시
- 접수중, 모집예정, 마감 순서의 결과 정렬과 과거 공고 제공
- 모바일 화면 대응
- 선택적 실공고 연동과 공고 프로필 적합도
- 공식 상세 페이지와 PDF·HWPX 첨부파일 수집
- 접수기간, 대상, 연령, 무주택, 소득·자산, 임대조건, 준비서류 구조화
- 공고문 분석 품질을 반영한 추천과 후속 질문
- 청약 가점·변동 추적·경쟁률 시범 모듈(화면 비활성)
- ICS 내부 기능
- SQLite 공고 누적과 오전 7시 증분 동기화 워크플로
- Git에 보존되는 검증된 실공고 스냅샷과 API 장애 폴백
- 프로필·관심 공고 저장
- 이전 추천 번호를 이용한 후속 질문

실공고 모드의 기본 범위는 LH·SH·GH 공공주택과 청약홈 APT 국민주택입니다. GH는 청약센터의 임대주택·매입임대 목록을 우선 수집하고 두 채널이 모두 실패할 때만 본사 게시판으로 대체합니다. 청약홈은 구조화된 주택구분이 국민인 공고만 수집·노출하고 민영 및 다른 청약홈 채널은 제외합니다. 최초 최근 90일을 적재하고 이후 매일 최근 7일을 중복 조회하며, 일요일에는 90일을 재대조합니다. 검증된 마지막 결과는 `data/live_housing_programs.json`에 저장해 Render가 초기화되거나 응답하지 않아도 Vercel 추천에 사용합니다. 이 스냅샷도 없을 때만 출품용 샘플 데이터를 사용합니다. 추천 카드는 모집 상태·추천 이유·신청기간을 표시하며 원문 추출 내용을 분석 결과나 지원내용으로 노출하지 않습니다. 즉석 원문 해석·통계 경쟁률·가점·변동 추적 UI도 검증 전까지 비활성입니다. 외부 LLM API와 완전한 RAG 검색은 아직 사용하지 않습니다. 문서에서 사용하는 에이전트 명칭은 역할을 설명하기 위한 논리적 구분이며 독립적으로 실행되는 다중 에이전트 시스템은 아닙니다.

## 실데이터 통합 기반

`services/announcement-api`에 독립 FastAPI 서비스를 추가하고 Next.js와 연결했습니다.

- 현재 샘플 JSON을 공통 공고 모델로 변환하는 소스
- 직접 수집 결과와 가점·매칭·원문·경쟁률·ICS·변동 추적 내부 모듈(원문·경쟁률 UI는 기본 비활성)
- 신청기간 기반 접수중, 모집예정, 마감, 일정 확인 필요 계산
- 데이터 출처와 일정 확인 여부 메타데이터
- SQLite 기반 프로필·관심 공고 저장

FastAPI 런타임은 별도 설치가 필요합니다. `npm run dev`에서는 저장 실공고를 사용하고 `npm run dev:full`에서는 기관에서 즉시 수집한 실공고를 사용합니다. Slack·Telegram 발송 기능은 서비스 범위에서 제외했습니다.

## 현재 처리 흐름

1. 사용자가 현재 상황을 문장으로 입력합니다.
2. `lib/parser`가 조건을 규칙 기반으로 추출합니다.
3. `lib/status`가 신청기간으로 모집 상태를 계산합니다.
4. `lib/matcher`가 사용자 조건과 각 공고의 점수 및 추천 이유를 계산합니다.
5. `lib/recommender`가 접수중, 모집예정, 마감 순으로 정렬합니다.
6. API가 결과를 반환하고 화면이 카드로 표시합니다.

## 향후 고도화

- LH·SH·GH 기관별 상세 파서 정확도와 회귀 테스트 확대
- 스캔 PDF·구형 HWP 및 표 구조 분석
- RAG 기반 공고 질의응답
- 사용자 피드백 기반 추천 개선
- 알림, 카카오톡, 모바일 앱 확장

공개적으로 관찰 가능한 기관별 공고 수집 패턴을 참고해 청나주 공통 모델에 맞게 직접 수집·분석 모듈을 재구성했습니다. LLM 기반 근거 검색은 후속 단계입니다. 자세한 구조는 `integration-architecture.md`, 단계별 구현 범위는 `development-backlog.md`에 정리돼 있습니다.

향후 상담 이력, 사용자 피드백, 신규 공고 데이터를 반영해 추천 품질을 고도화할 수 있는 구조를 지향합니다. 시스템이 지속적으로 스스로 학습한다고 전제하지 않으며, 데이터 반영과 평가 절차를 거쳐 개선하는 방향입니다.

## 이용 범위

추천 결과는 공모전 시연과 정보 탐색을 위한 참고용입니다. 실제 신청 자격을 확정하지 않으며, 사용자는 반드시 해당 기관의 최신 공고 원문을 확인해야 합니다.
