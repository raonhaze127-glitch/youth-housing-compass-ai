# 프로젝트 개요

## 한 줄 소개

청년주거나침반 AI는 사용자가 입력한 주거 조건을 규칙으로 구조화하고, 샘플 공공주거 데이터와 비교해 참고할 사업을 보여주는 공모전 출품용 MVP입니다.

## 현재 구현

- 한 번의 자연어 문장을 받는 대화형 입력 UI
- 규칙 기반 나이, 지역, 무주택 여부, 소득 수준, 관심 유형 추출
- 구조화된 `housing_programs.json` 샘플 데이터 기반 추천
- 추천 이유, 신청기간, 지원내용, 원문 링크를 담은 결과 카드
- 신청기간을 기준으로 계산한 접수중, 모집예정, 마감 상태 표시
- 접수중, 모집예정, 마감 순서의 결과 정렬과 과거 공고 제공
- 모바일 화면 대응
- 선택적 실공고 연동과 공고 프로필 적합도
- 청약 가점·자격 사전 점검
- 원문 섹션 해석, 경쟁률, ICS, 변동 추적
- 프로필·관심 공고 저장
- 이전 추천 번호를 이용한 후속 질문

샘플 모드는 외부 시스템을 호출하지 않습니다. 실공고 모드는 k-apt-alert 호환 API를 호출하며 현재 공고 해석은 원문 추출 섹션과 결정론 규칙을 사용합니다. 외부 LLM API와 완전한 RAG 검색은 아직 사용하지 않습니다. 문서에서 사용하는 에이전트 명칭은 역할을 설명하기 위한 논리적 구분이며 독립적으로 실행되는 다중 에이전트 시스템은 아닙니다.

## 실데이터 통합 기반

`services/announcement-api`에 독립 FastAPI 서비스를 추가하고 Next.js와 연결했습니다.

- 현재 샘플 JSON을 공통 공고 모델로 변환하는 소스
- k-apt-alert 호환 API 응답과 가점·매칭·원문·경쟁률·ICS·변동 추적 어댑터
- 신청기간 기반 접수중, 모집예정, 마감, 일정 확인 필요 계산
- 데이터 출처와 일정 확인 여부 메타데이터
- SQLite 기반 프로필·관심 공고 저장

FastAPI 런타임은 별도 설치가 필요합니다. `npm run dev`에서는 샘플 데이터를 사용하고 `npm run dev:full`에서는 실공고 데이터를 사용합니다. Slack·Telegram 발송 기능은 서비스 범위에서 제외했습니다.

## 현재 처리 흐름

1. 사용자가 현재 상황을 문장으로 입력합니다.
2. `lib/parser`가 조건을 규칙 기반으로 추출합니다.
3. `lib/status`가 신청기간으로 모집 상태를 계산합니다.
4. `lib/matcher`가 사용자 조건과 각 공고의 점수 및 추천 이유를 계산합니다.
5. `lib/recommender`가 접수중, 모집예정, 마감 순으로 정렬합니다.
6. API가 결과를 반환하고 화면이 카드로 표시합니다.

## 향후 고도화

- 기관별 수집기를 청나주가 직접 운영하는 실데이터 수집
- 공고 원문에서 자격요건, 신청기간, 지역, 공급대상, 준비서류 분석
- RAG 기반 공고 질의응답
- 사용자 피드백 기반 추천 개선
- 알림, 카카오톡, 모바일 앱 확장

k-apt-alert의 수집·분석 기능은 호환 API 경계로 통합했습니다. 기관별 수집기를 청나주가 직접 운영하는 구조와 LLM 기반 근거 검색은 후속 단계입니다. 자세한 구조는 `integration-architecture.md`, 단계별 구현 범위는 `development-backlog.md`에 정리돼 있습니다.

향후 상담 이력, 사용자 피드백, 신규 공고 데이터를 반영해 추천 품질을 고도화할 수 있는 구조를 지향합니다. 시스템이 지속적으로 스스로 학습한다고 전제하지 않으며, 데이터 반영과 평가 절차를 거쳐 개선하는 방향입니다.

## 이용 범위

추천 결과는 공모전 시연과 정보 탐색을 위한 참고용입니다. 실제 신청 자격을 확정하지 않으며, 사용자는 반드시 해당 기관의 최신 공고 원문을 확인해야 합니다.
