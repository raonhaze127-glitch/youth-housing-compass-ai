# 로드맵

## 현재: 출품용 MVP

- 규칙 기반 사용자 조건 추출
- 구조화된 샘플 공고 데이터
- 모집 상태 자동 계산 및 정렬
- 추천 카드와 모바일 화면

## 1단계: 실공고 수집 기반

- k-apt-alert 참고 구현의 출처와 사용 범위 확인
- Python/FastAPI 공고 수집 서비스 분리 (초기 골격 완료)
- 샘플·기관 직접 수집 소스와 화면 연결 (완료)
- 가점·매칭·ICS·변동 추적 내부 모듈 연결 (완료)
- 청약홈, LH, SH, GH 실데이터 수집
- 기관별 원문 저장과 공통 스키마 정규화
- 샘플 데이터와 실데이터 모드 분리

## 2단계: 원문 분석과 판정

- 공고 원문 분석 및 출처 검증
- HTML, PDF, HWP/HWPX 추출
- 신규·수정·마감·삭제 변동 추적
- 청약 가점과 청년주거 사업별 자격 규칙
- 충족·불충족·확인 필요 판정과 규칙 버전 관리

직접 수집 기반 변동·청약 가점 연결은 완료했습니다. 원문 해석은 시범 추출기를 보존하되 화면에서 숨겼으며, 공고별 구조화와 근거 검증을 거쳐 다시 연결합니다. 청년 임대·월세지원 사업별 세부 규칙과 규칙 버전 저장은 계속 확장합니다.

## 3단계: 사용자 경험 고도화

- 원문 근거를 표시하는 RAG 기반 공고 질의응답
- 상담 이력과 명시적 사용자 피드백을 활용한 추천 평가
- 경쟁률, 일정, ICS, 즐겨찾기
- 신규·변경 공고 알림 및 카카오톡 연계
- 모바일 앱 확장

현재 ICS·즐겨찾기와 후속 질문 대화는 구현했습니다. 경쟁률은 LH·SH·GH 공식 모집결과 데이터 연결 전까지 화면에서 숨깁니다. Slack·Telegram 발송은 서비스 범위에서 제외했습니다. 카카오톡과 RAG 검색은 별도 검토가 필요한 후속 작업입니다.

각 단계는 데이터 정확도와 사용자 검증을 선행합니다. 자동 학습을 전제하지 않으며, 수집 데이터와 피드백을 검토해 추천 품질을 개선합니다.

상세 설계는 `integration-architecture.md`, 작업 단위와 완료 조건은 `development-backlog.md`를 기준으로 합니다.
