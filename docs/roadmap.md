# 로드맵

## 현재: 출품용 MVP

- 규칙 기반 사용자 조건 추출
- LH·SH·GH 및 청약홈 국민주택 저장 실공고와 샘플 최종 폴백
- 실제 모집공고 필터와 PDF·HWPX 공고문 구조화
- 모집 상태 자동 계산 및 정렬
- 추천 카드와 모바일 화면

## 1단계: 실공고 수집 기반

- 참고 수집 구조의 출처와 사용 범위 확인
- Python/FastAPI 공고 수집 서비스 분리 (초기 골격 완료)
- 샘플·기관 직접 수집 소스와 화면 연결 (완료)
- 가점·매칭·ICS·변동 추적 내부 모듈 연결 (완료)
- LH·SH·GH 공공주택 실데이터 수집 (완료)
- 청약홈 APT 국민주택 수집 및 민영주택 제외 (완료)
- 검증 실공고 스냅샷과 Render 장애 폴백 (완료)
- 청약홈 민간주택 수집은 향후 확장
- 기관별 원문 분석 결과와 공통 스키마 정규화 (1차 완료)
- 샘플 데이터와 실데이터 모드 분리 (완료)

## 2단계: 원문 분석과 판정

- 공고 원문 분석 및 출처 검증 (HTML·PDF·HWPX 1차 완료)
- 스캔 PDF·구형 HWP와 복잡한 표 추출 보완
- 신규·수정·마감·삭제 변동 추적
- 청약 가점과 청년주거 사업별 자격 규칙
- 충족·불충족·확인 필요 판정과 규칙 버전 관리

공고문 구조화 결과는 추천 카드와 후속 질문에 연결했습니다. 변동 추적·청약 가점·경쟁률은 내부 시범 모듈만 보존하고 화면에서는 숨겼습니다. 청년 임대·월세지원 사업별 세부 규칙과 규칙 버전 저장은 계속 확장합니다.

## 3단계: 사용자 경험 고도화

- 원문 근거를 표시하는 RAG 기반 공고 질의응답
- 상담 이력과 명시적 사용자 피드백을 활용한 추천 평가
- 경쟁률, 일정, ICS, 즐겨찾기
- 신규·변경 공고 알림 및 카카오톡 연계
- 모바일 앱 확장

현재 ICS·즐겨찾기와 후속 질문 대화는 구현했습니다. 경쟁률은 LH·SH·GH 공식 모집결과 데이터 연결 전까지 화면에서 숨깁니다. Slack·Telegram 발송은 서비스 범위에서 제외했습니다. 카카오톡과 RAG 검색은 별도 검토가 필요한 후속 작업입니다.

각 단계는 데이터 정확도와 사용자 검증을 선행합니다. 자동 학습을 전제하지 않으며, 수집 데이터와 피드백을 검토해 추천 품질을 개선합니다.

상세 설계는 `integration-architecture.md`, 작업 단위와 완료 조건은 `development-backlog.md`를 기준으로 합니다.
