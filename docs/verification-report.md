# 통합 검증 기록

기준일: 2026-06-20

## 확인된 범위

| 항목 | 검증 방법 | 결과 |
|---|---|---|
| 모집 상태 계산·정렬 | Python 단위 테스트와 샘플 통합 검사 | 통과 |
| 공고 수집·추천·후속 질문 | Next.js와 FastAPI 샘플 통합 검사 | 통과 |
| 자격·가점, 적합도, ICS, 변동 추적 | 직접 구현 단위·API 검사 | 통과 |
| 프로필·즐겨찾기 저장 | Python 단위 테스트 | 통과 |
| 프로덕션 빌드와 타입 검사 | `npm run build` | 통과 |

Python 표준 테스트와 Next.js 프로덕션 빌드를 통과했습니다. 기존 호환 API 검증 이력과 별도로 LH·SH·GH 직접 수집, 청약홈 APT 국민주택 수집, 민영주택의 수집·저장 조회·변동 이력·정적 스냅샷 차단, 내부 가점·매칭·ICS·변동 추적을 확인했습니다. 원문 규칙 추출과 통계 경쟁률 API는 시범 코드만 보존하고 카드에서는 기본 비활성화했습니다. SQLite 공고 upsert, 내용 해시 변경 감지, 동기화 상태 저장, 증분 누적과 수동 동기화 API도 단위 검사에 포함했습니다.

Render 무료 인스턴스 초기화 후에도 추천을 유지하기 위해 검증된 실공고 82건의 정적 스냅샷을 생성했습니다. 필수 필드, 기관 범위, 중복 ID, 최소 건수를 검사한 뒤에만 파일을 교체하며 실시간 API가 실패하면 이 스냅샷을 사용합니다.

## 외부 의존 상태

외부 공고 프록시 없이 운영 배포에서 과거 청약홈·LH·SH·GH 총 135건을 확인한 이력이 있습니다. 현재 기본 범위에서는 LH·SH·GH와 청약홈 APT 중 주택구분이 국민인 공고만 새로 수집·노출합니다. 기관별 실패가 전체 조회 장애로 이어지지 않도록 부분 성공과 마지막 성공 저장 결과를 사용합니다. 로컬 실수집 재검사는 실행 환경의 외부 네트워크 차단으로 완료하지 못했으며, 배포 후 예약 동기화 응답으로 다시 확인해야 합니다.

Slack·Telegram 발송 기능은 서비스 범위에서 제외했으며 관련 UI·API·워커·환경변수·테스트를 제거했습니다.

## 해석 주의

통과 결과는 API 계약과 MVP 흐름이 동작한다는 의미입니다. 실제 신청 자격을 확정하거나 외부 공고 제공기관의 상시 가용성을 보장하지 않습니다.
